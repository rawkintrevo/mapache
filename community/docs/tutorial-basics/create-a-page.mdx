# Create a Workspace

A workspace is the project boundary for a repository, auth settings, file sync,
environment variables, and the materialized home folder. Treat it like a
disposable development environment with a stable connection to the code you care
about.

## Before You Start

- Have the Git repository URL ready.
- Decide which credentials the workspace should be allowed to use.
- Pick the runtime image that best matches the project.

## Create the Workspace

1. Open Mapache Tools.
2. Select **New Workspace**.
3. Paste the repository URL.
4. Confirm the source summary.
5. Save the workspace.

The workspace appears in the left drawer. From there you can launch sessions,
refresh repository state, and inspect generated files.

Each launched session starts its own Pi conversation thread. Starting another
session in the same workspace reuses the workspace files and home folder, but it
does not merge Pi chat history with the earlier session.

## Keep It Scoped

Create separate workspaces when projects need different credentials, dependency
sets, or agent skills. Sharing one workspace across unrelated projects makes it
harder to reason about auth exposure and runtime drift.
