# Maintain the Community Site

The community site is a Docusaurus package under `community/`. Keep it visually
aligned with the webapp and keep operational copy focused on Mapache workflows.

## Local Development

```bash
cd community
npm run start
```

## Production Build

```bash
cd community
npm run build
```

The build emits static files under `community/build`. Broken links are treated
as build failures, so update routes and footer links together.

## Style Rules

- Reuse the Mapache tokens in `community/src/css/custom.css`.
- Prefer compact panels, 6-8px radii, and developer-tool density.
- Avoid starter Docusaurus illustrations and generic tutorial language.
- Keep docs changes narrow and connected to real product behavior.
