Mapache starts from a practical premise: coding agents are easier to trust when
each project has a clear boundary.

{/* truncate */}

A workspace ties together a repository, runtime image, session history, and
explicitly enabled credentials. That boundary keeps project-specific auth,
dependency state, and generated files from leaking into unrelated work.

The goal is not to hide the terminal. It is to make the terminal safer to use by
placing it inside a serverless environment that was created for the project in
front of you.
