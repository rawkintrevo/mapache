Mapache treats the terminal as the primary interface for serious agent work.
The webapp exists to make that terminal easier to provision, inspect, and shut
down.

{/* truncate */}

That is why the app layout is dense: a workspace drawer on the left, a session
surface in the center, and an inspector on the right. The surrounding UI should
answer operational questions quickly:

- Which repository is attached?
- Which session image is running?
- Which credentials are enabled?
- Which files changed?
- Which command verified the result?

The community site now follows the same visual grammar so documentation feels
like part of the tool instead of a separate marketing skin.
